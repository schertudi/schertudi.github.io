DS Polkka

In this project, I wanted to experiment with the concept of a "dynamic" rhythm game - one that feels like it really responds to the player's actions. To this extent, I made a lot of animations that activate on correct hitting of beats. I also have a system where gaining a perfect combo in some sections activates a more interesting piece of music. My hope is that this would make the game much more engaging than typical rhythm games.

I have playtested this with a few people and the reactions were very positive, so I believe I am on the right track.

At the moment, I have a basic gameplay loop where players can play through one short song. In the future I plan on extending this to include a tutorial for new players, multiple songs, and score tracking as well as a bigger variety of animations and input methods.

I was interested in developing for the Nintendo DS as it requires additional challenges over a typical desktop project. Performance and space are more of an issue and it can be tricky finding comprehensive documentation on features. However, it has been a very fun project so far!

My work

This project was built off devkitPro's system libraries for the Nintendo DS, plus a wrapper library to simplify rendering 2d graphics. This allowed me to draw to the screen, read input, perform calculations and play sound without much effort. All functionality beyond that is coded by myself:
  • Render custom shapes and animations by drawing triangles onto the screen
  • Spawn hit circles in time to the music, and determine whether or not the player has correctly hit these
  • A flexible animation system that makes it trivial to create new animations that react to player behaviour. This can be extended for complex animations involving multiple hit circles.
  • A parser to quickly define level data. This includes information about where and when hit circles are spawned, and the ability to add and customize animations that react to those circles being hit.
  • The player can "level up" during a song by correctly hitting a sequence of blue beats. This allows additional animations and a backing song track to play.
  • Optimised to ensure a stable 60fps on a 67mhz processor* and that memory use never exceeds the DS's 4mb of ram.
*the DS does have a second CPU, but it can't do much.

You can view the source code for this project at github.com/schertudi/ds-rhythm-game.
